02 — Problem & market

Every AI system is deployed
without a safety layer.

The core problem

When companies deploy AI — a customer support bot, a coding assistant, a document analyser — they pipe user inputs directly to the LLM with zero inspection. The LLM responds and the output goes back to the user, again with zero inspection.

This creates three catastrophic attack surfaces simultaneously:

IN
Malicious input (prompt injection)
Attacker crafts a prompt that overrides the AI's instructions, causing it to ignore safety rules, leak data, or perform unauthorized actions.
CTX
Contaminated context (indirect injection)
Malicious instructions hidden in documents, emails, or web pages that AI agents read — invisible to users, but interpreted as commands by the model.
OUT
Dangerous output (data leakage)
LLM returns API keys, PII, internal system prompts, or harmful instructions that should never reach the user.

Real breach evidence (2025–2026)

EchoLeak — Microsoft 365 Copilot 2025

Zero-click prompt injection via email. Hidden instructions caused Copilot to silently exfiltrate enterprise data. No user action required.

GitHub Copilot CVE-2025-53773 CVSS 9.6

Prompt injection in pull request descriptions → remote code execution with system privileges. 85%+ attack success rate against existing defenses.

Samsung source code leak 2023

Engineers pasted proprietary semiconductor code into ChatGPT. Data left Samsung's perimeter permanently. ChatGPT banned company-wide.

OpenClaw MCP crisis Feb 2026

135k-star OSS agent framework. 1,184 malicious MCP tool skills injected. 21,000 exposed instances. Credentials stolen via poisoned tool registry.

05 — Competitive moat

Five compounding
advantages.

01
Detection corpus compounds with every customer
Every blocked attack gets anonymized and added to the training corpus for the ML classifier. Month 1: 97 injection samples. Month 18: 500,000+ real-world attacks. The classifier accuracy improves with scale in a way no competitor can replicate without our data. Lakera built 80M data points from Gandalf — that data was worth $300M at exit. We start building ours on day one.
02
MCP security is a first-mover opportunity
The Model Context Protocol is now the standard for how AI agents call external tools. It's used by Claude, GPT-4o, Gemini, and every major agent framework. The OpenClaw crisis (Feb 2026) exposed 21,000 vulnerable instances. Nobody has an open-source MCP security gateway. We build it in Phase 1 and become the default reference implementation before anyone else realizes the opportunity.
03
Switching costs are embedded in the infrastructure
Once a company routes their AI traffic through ShieldAI, the dashboard becomes their AI audit trail, the threat data is their compliance evidence, and the rules are tuned to their specific use case. Switching means losing months of threat intelligence and rebuilding their compliance documentation from zero. Infrastructure-layer software has 80–90% gross margins and near-zero churn for exactly this reason.
04
Open source creates distribution no budget can buy
Lakera spent $23M in VC to acquire users. Our GitHub repo, published research reports, and Injection Playground create inbound users for free. A well-timed HackerNews post drives 10,000 visitors in 48 hours. A useful open-source tool stars organically. A security research report gets cited in articles. The open-core model converts developer trust into enterprise contracts — that trust cannot be purchased at any price.
05
Compliance integration is an enterprise lock-in
The SOC2 audit logs, HIPAA evidence packs, and GDPR processing records we generate for enterprise customers are formal legal documentation. Once an enterprise uses ShieldAI's logs in their annual SOC2 audit, switching vendors means re-doing that audit with a new vendor's tooling. Compliance is the stickiest enterprise product category in software — ask any SIEM vendor.
